Post-v0.4.3 — Bug fixes & CI hardening

2026-06-02 – 2026-06-03 · unreleased on main

Overview

Two installability bugs fixed and the docs deploy hardened. No public API changes — every importable symbol behaves identically to v0.4.3. A v0.4.4 tag would publish these fixes to PyPI.

Bug fixes

  • statsmodels pin corrected ([test-parity] / [all-extras]): statsmodels>=0.15 → >=0.14. statsmodels has never released a 0.15 version (latest is 0.14.x); the original pin made both groups uninstallable — pip's resolver failed before any test ran. The floor >=0.14 is correct because nstat.extras.validation.statsmodels_bridge only calls statsmodels.api.GLM, available throughout the 0.14 line. (PR #132)
  • Clusterless bridge aligned with replay_trajectory_classification>=1.4 (nstat.extras.decoding.clusterless_bridge):
    • ClusterlessDecoder takes singular environment= / transition_type= arguments in 1.4; the prior bridge passed plural names (environments= / continuous_transition_types=), raising TypeError.
    • ClusterlessClassifier.continuous_transition_types requires a square (n_states × n_states) matrix; the prior bridge passed a single 1 × n_states row, causing an index-out-of-bounds error in upstream's transition-matrix assembly. The bridge now builds the full square matrix (per-state movement model on the diagonal, uniform spatial transition off-diagonal).
    The [clusterless] dep floor is bumped to >=1.4. (PR #132)

Infrastructure

  • intersphinx fallback inventories vendored under docs/_inv/ (numpy, scipy, matplotlib, sympy, Python). Each mapping tries the live inventory first and falls back to the committed copy on network failure, with intersphinx_timeout=10. Prevents transient host outages (observed during the v0.4.3 GitHub Pages deploy on docs.scipy.org) from failing the strict -W build and blocking the publish. A make refresh-intersphinx-inv target (tools/refresh_intersphinx_inv.py) re-pulls the vendored copies to keep them current. (PR #130)

Documentation

  • README and nstat/extras/__init__.py synced with the v0.4.0 feature surface:
    • Clusterless bridge row added to the "Related Python projects" table in the README.
    • nstat.extras module docstring updated from "Three subpackages" to "Five" (the em and decoding subpackages added in v0.4.0 were not counted).
    • Stale "foundation for unported KF_EM / PP_EM / mPPCO_EM" wording removed from the Dynamax README row — that family is shipped.
    (PR #126 / PR #129)
  • Brand reskin attempted and reverted: a docs reskin (oscilloscope / lab-instrument aesthetic on top of sphinx_rtd_theme) was merged in PR #131 then reverted in PR #133. The prior blue Read-the-Docs theme is restored.
