Post-v0.4.3 hotfixes

2026-06-03 · PRs #132 and #133 merged to main — no version bump
These fixes landed on main after the v0.4.3 tag. They will be included in the next tagged release. No public API changes.

Fix 1 — statsmodels dependency pin corrected (PR #132)

The [test-parity] and [all-extras] optional-dependency groups previously declared statsmodels>=0.15. Because statsmodels 0.15 was never released (latest is 0.14.x), pip’s resolver failed immediately — making those install groups silently uninstallable. The pin has been corrected to statsmodels>=0.14, which is the actual floor required by the bridge.

This also means the v0.4.1 release note that described this pin was historically inaccurate: it documented an intended pin that shipped broken and was never resolvable by pip.

Fix 2 — Clusterless bridge aligned with replay_trajectory_classification 1.4 (PR #132)

The clusterless bridge (nstat.extras.decoding.clusterless_bridge) was constructed with argument names that do not match the replay_trajectory_classification>=1.4 (installs as 1.4.1) API, breaking the extras-clusterless CI job:

  • ClusterlessDecoder takes singular environment= / transition_type= keyword arguments. The bridge was incorrectly passing the plural forms (environments= / continuous_transition_types=) used by the classifier, raising TypeError.
  • ClusterlessClassifier.continuous_transition_types must be a square (n_states × n_states) matrix. The bridge was passing a single 1×N row, so upstream’s transition-matrix assembly indexed out of bounds. The fix builds the full square matrix: per-state movement model on the diagonal, uniform spatial transition off-diagonal — matching the upstream default layout.

Both previously-failing functional tests in tests/extras/test_clusterless_bridge.py now pass against replay_trajectory_classification 1.4.1.

Fix 3 — Docs theme revert (PR #133)

The experimental oscilloscope / dark lab-instrument Sphinx theme introduced in PR #131 was reverted in full. The prior blue Read-the-Docs stylesheet is restored across docs/_static/custom.css, docs/conf.py, and docs/extras_summary.html.
