2026-06-03 — CI / installability fixes (post-v0.4.3)

2026-06-03 · PRs #132 & #133 · no version bump

Overview

Two installability/CI bugs that slipped into main after the v0.4.3 tag, fixed on 2026-06-03. No public API changes. These fixes will ship in the next tagged release.

Fix 1 — Correct statsmodels pin (PR #132)

The [test-parity] and [all-extras] optional-dep groups carried statsmodels>=0.15, but statsmodels never shipped a 0.15 release (latest is 0.14.x). pip's resolver rejected the constraint outright, making both groups completely uninstallable in the v0.4.1–v0.4.3 window.

  • Corrected to statsmodels>=0.14 in pyproject.toml. The statsmodels bridge only uses statsmodels.api.GLM, available throughout 0.14.
  • The v0.4.1 release note that described adding this pin is updated separately; the original intent (avoid a scipy _lazywhere symbol removal) is satisfied by 0.14.4+, which shipped the fix before 0.15 was ever released.

Fix 2 — Align clusterless bridge with replay_trajectory_classification 1.4 (PR #132)

The clusterless bridge (nstat.extras.decoding.clusterless_bridge) constructed the upstream decoder/classifier with argument names that changed between 1.3 and 1.4:

  • ClusterlessDecoder takes singular environment= / transition_type=, not the plural names used by the classifier. Passing the plural names raised TypeError.
  • ClusterlessClassifier.continuous_transition_types must be an n×n square matrix; the bridge passed a single 1×n row, causing an out-of-bounds index during transition-matrix assembly. Fixed by building the full square matrix (per-state movement model on the diagonal, uniform spatial transition off-diagonal).

Verified against replay_trajectory_classification 1.4.1: the two previously-failing functional tests in tests/extras/test_clusterless_bridge.py now pass.

Fix 3 — Revert experimental docs reskin (PR #133)

An oscilloscope-themed Sphinx stylesheet (PR #131, same day) was merged and immediately reverted after it clashed with the existing hand-built landing pages. The blue RTD theme from v0.4.3 is restored. No user-visible content change.
