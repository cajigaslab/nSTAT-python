Post-v0.4.3 CI + Docs Fixes

2026-06-03 — Two follow-up PRs merged to main after the v0.4.3 tag: a critical extras dependency fix (PR #132) and a docs theme revert (PR #133). No public API changes.

Action needed for PyPI users on v0.4.3: The statsmodels>=0.15 pin introduced in v0.4.1 refers to a version that was never released (latest is 0.14.6). This makes pip install nstat-toolbox[test-parity] and pip install nstat-toolbox[all-extras] fail with a resolver error on any v0.4.x PyPI release. The fix (>=0.14) is on main as of this PR; a v0.4.4 patch tag will ship it to PyPI.

PR #132 — Extras dependency fixes (merged 2026-06-03)

Fix 1: correct statsmodels pin to >=0.14

statsmodels has never released a 0.15 (current latest: 0.14.6). The >=0.15 floor introduced in v0.4.1 meant pip's dependency resolver could not find a satisfying candidate, so the [test-parity] and [all-extras] install groups were entirely uninstallable. The floor is now >=0.14, which covers the whole 0.14 line (0.14.0 – 0.14.6). The bridge only calls statsmodels.api.GLM, available throughout 0.14.

Fix 2: align clusterless bridge with replay_trajectory_classification 1.4 API

The nstat.extras.decoding.clusterless_bridge passed constructor arguments that no longer match replay_trajectory_classification>=1.4 (which installs as 1.4.1):

  • ClusterlessDecoder takes singular environment= / transition_type=, not the plural forms (environments= / continuous_transition_types=) used by the classifier — passing the plural names raised TypeError.
  • ClusterlessClassifier.continuous_transition_types must be a square (n × n) matrix; the bridge passed a single 1 × n row, so the upstream transition-matrix assembly indexed out of bounds. The bridge now builds the full square matrix: per-state movement model on the diagonal, uniform spatial transition off-diagonal (matching the upstream default layout).

The two previously-failing functional tests in tests/extras/test_clusterless_bridge.py now pass against replay_trajectory_classification 1.4.1.

PR #133 — Revert docs reskin (merged 2026-06-03)

PR #131 (merged 2026-06-03) restyle the Sphinx docs site with a phosphor-green oscilloscope theme (docs/_static/custom.css + docs/conf.py changes). PR #133 reverts this, restoring the prior blue Read-the-Docs theme. The custom CSS and conf.py additions from PR #131 are removed; the style_nav_header_background RTD theme option returns to #2c5282 (nSTAT brand blue).

No API or behavior changes.
